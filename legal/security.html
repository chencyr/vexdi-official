資安宣告

本宣告說明 Vexdi 維思迪在資料保護、帳號驗證、程式碼安全、部署、事件回應與漏洞通報上的承諾與實作。資安是持續過程，不是一次性檢查清單。

最後更新 2026 · 04 · 22 適用於 本網站及所有客戶專案

我們的態度：客戶資料是負債、不是資產。能不存就不存、能少權限就少權限、能加密就加密。所有專案上線前必跑一次第三方靜態掃描與 OWASP Top 10 檢核。

01資料保護原則

  • 最小蒐集：只蒐集完成服務所必要的資料；可匿名處理的就匿名處理。
  • 最小權限：專案參與者僅取得執行該任務所需的權限；不發放共享帳號、不長期保留 root key。
  • 最短保留：專案結束後 30 日內刪除我方持有之客戶資料副本，除非客戶要求保留。
  • 透明可追溯：所有對客戶系統的變更記錄於 git history 與部署日誌，可隨時稽核。

02帳號與身份驗證

  • 所有內部與雲端服務帳號強制啟用 多因素驗證（MFA），且使用硬體 Key（YubiKey）或 TOTP，不使用 SMS。
  • 密碼使用 1Password 等具零知識架構之密碼管理員產生與儲存，不以明文形式於 Slack / Email / 文件中傳遞。
  • 客戶帳號（GitHub、AWS、Vercel 等）權限交付時即移轉所有權給客戶指定主帳號，我方僅保留必要協作者權限，並於專案結束時主動移除。

03程式碼安全

  • 原始碼管理：使用 GitHub 私有 repository，所有 commit 簽署 GPG。
  • 依賴掃描：使用 Dependabot / Snyk 持續監測第三方套件漏洞，CRITICAL 級別 24 小時內處理。
  • 機敏資訊：不將 API Key、密碼、token 寫入原始碼；統一以環境變數或專用 secret manager（Vercel Env、AWS Secrets Manager）管理。
  • Code Review：所有 production-bound merge 至少經過一次 code review。
  • 上線前掃描：跑一次靜態分析（CodeQL / ESLint security rules）與 OWASP Top 10 檢核清單。

04資料傳輸與儲存

  • 傳輸層：所有對外服務強制 HTTPS（TLS 1.2+），HSTS 設置至少 6 個月。
  • 儲存層：資料庫、物件儲存預設啟用 at-rest encryption（AES-256）；客戶可指定金鑰管理方案。
  • 備份：關鍵資料於異地備援，備份檔同樣加密；備份保留 30 日。
  • 個資欄位：密碼以 bcrypt / argon2 雜湊；身份證、信用卡等高敏欄位使用獨立 KMS 加密或委由合規金流商處理（如 TapPay、ECPay），不自存信用卡卡號。

05部署與基礎建設

  • 預設部署於 Vercel / Cloudflare / AWS（依專案性質選擇）— 均通過 SOC 2 Type II 認證。
  • 對外服務前置 Cloudflare WAF 與 rate limiting，阻擋常見攻擊向量。
  • 後台管理介面僅允許從特定 IP 範圍或 VPN 存取（依專案需求）。
  • 定期更新作業系統與 runtime 版本，不留長期未維護之服務。

06員工與承包商

  • 目前團隊極小（創辦人 + 必要時邀請之合作工程師），所有人簽署 NDA 後始接觸客戶資料。
  • 離職或合作結束時 24 小時內撤銷所有存取權限。
  • 個人裝置（筆電）強制全碟加密（FileVault / BitLocker）並啟用螢幕鎖。

07事件回應

若發生資安事件（資料外洩、未授權存取、嚴重漏洞），我們將：

  1. 1 小時內啟動內部處理並隔離受影響系統
  2. 24 小時內通知受影響客戶並提供初步影響評估
  3. 72 小時內提供詳細事件報告（時間軸、影響範圍、修復措施、後續預防）
  4. 依個資法規定協助客戶向主管機關通報

08漏洞通報（Responsible Disclosure）

如您發現本網站、本公司營運系統、或我們交付給客戶之專案存在資安漏洞，歡迎透過下列方式通報，我們承諾不對善意通報者採取法律行動：

  • 聯絡：security@vexdi.com（暫時與 hi@vexdi.com 共用，主旨請註明 [SECURITY]）
  • 請提供：漏洞描述、重現步驟、潛在影響、您的聯絡方式
  • 請避免：大規模掃描、社交工程攻擊、影響服務穩定之測試
  • 我們承諾：72 小時內回覆、合理時程內修復、修復後致謝（除非您希望匿名）

09第三方稽核

目前公司規模尚未達到取得 ISO 27001 / SOC 2 等正式認證之階段，但我們的內部資安操作參照上述標準執行。對於有合規需求之企業客戶，我們可協助：

  • 填寫您的供應商資安問卷
  • 簽署資料處理協議（DPA）
  • 協調滲透測試（由您指定第三方執行，費用另計）

10持續改進

資安是一個持續過程，不是一次性檢查清單。我們會定期回顧本宣告與內部操作，並隨產業最佳實踐調整。重大更新將於本頁公告。
